Methodology & data notes

  • Trade aggregation: totals, win rate, ratios, expectancy, and yearly results sum the selected closed-trade rows. Exact duplicate rows inside a source are removed. WPBR is preferred; legacy PBR is only a fallback, never an additional system.
  • Curve: ACTIVE curve: realized-P&L equity by exit date. A compatible regular equity file was not available for every active system, so no mark-to-market values were invented.
  • Realized-curve limitation: when used, P&L appears only on exit dates. It captures chronological realized drawdown but not intratrade mark-to-market drawdown. Entry/exit overlap is used for concurrency and capital usage; no daily prices are invented.
  • Capital basis: trade notional is inferred as |P&L dollars ÷ P&L %| (RL native exports use the engine's $47,500 sizing). The normalized denominator is observed peak concurrent gross notional, not a claim about a broker account balance.
  • Annualized return: CAGR of raw total P&L divided by that estimated capital basis over the available date span. PPCD is profit per calendar day over the same span.
  • Yearly DD: realized P&L drawdown reset at each calendar year's first event, regardless of the main curve source.
